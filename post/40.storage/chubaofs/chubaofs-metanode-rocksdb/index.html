MetaNode RocksDB 存储引擎测试说明

基本功能测试

测试rocksdb metanode chubaofs 的基本功能,确保rocskdb metanode 基本功能的正常.

启动

测试metanode rocksdb 引擎chubaofs集群启动功能是否正常.

测试步骤:

  1. 部署 chubaofs rockdb 待测试版本集群；
  2. metanode 节点配置文件中storeType:1, 设置所有 metanode 存储引擎为 rocksdb 存储模式；
  3. 启动 chubaofs 集群，检查是否能正常启动；

创建 vol

测试metanode rocksdb 引擎chubaofs集群创建vol是否正常.

测试步骤:

  1. 部署 chubaofs rockdb 待测试版本集群；
  2. metanode 节点配置文件中storeType:1, 设置所有 metanode 存储引擎为 rocksdb 存储模式；
  3. 启动 chubaofs 集群;
  4. 调用createVolAPI, 创建不同 vol，检查是否创建成功；
  5. 使用客户端挂载新创建的 vol，检查 vol 是否挂载成功；

删除 vol

测试metanode rocksdb 引擎chubaofs集群删除vol是否正常.

测试步骤:

  1. 部署 chubaofs rockdb 待测试版本集群；
  2. metanode 节点配置文件中storeType:1, 设置所有 metanode 存储引擎为 rocksdb 存储模式；
  3. 启动 chubaofs 集群;
  4. 调用deleteVolAPI, 删除一个已存在的 vol，检查删除操作是否成功;

创建目录

测试metanode rocksdb 引擎chubaofs集群创建目录是否正常.

测试步骤:

  3. 启动 chubaofs 集群;
  4. 挂载 vol(若无 vol，则创建一个)；
  5. 在 vol 中执行创建目录操作，检查目录是否创建成功；

测试metanode rocksdb 引擎chubaofs集群删除目录是否正常.

  4. 挂载 vol(若无 vol，则创建一个)；
  5. 删除 vol 中的一个已存在目录，检查目录是否删除成功；

稳定性测试

测试metanode rocksdb 引擎chubaofs集群的稳定性.

metanode 节点故障模拟

测试metanode rocksdb 引擎chubaofs集群metanode节点故障时,集群功能是否正常.

测试步骤:

  1. 部署 chubaofs rockdb 待测试版本集群；
  2. metanode 节点配置文件中storeType:1, 设置所有 metanode 存储引擎为 rocksdb 存储模式；
  3. 启动 chubaofs 集群;
  4. 挂载 vol(若无 vol，则创建一个)；
  5. 在 vol 中一直执行文件读写操作；
  6. 随机间隔一段时间，随机选择一个 metanode 结束运行(kill)；
  7. 检查集群文件读写是否正常；

测试metanode rocksdb 引擎chubaofs集群datanode节点故障时,集群功能是否正常..

测试步骤:

  1. 部署 chubaofs rockdb 待测试版本集群；
  2. metanode 节点配置文件中storeType:1, 设置所有 metanode 存储引擎为 rocksdb 存储模式；
  3. 启动 chubaofs 集群;
  4. 挂载 vol(若无 vol，则创建一个)；
  5. 在 vol 中一直执行文件读写操作；
  6. 随机间隔一段时间，随机选择一个 datanode 结束运行(kill)；
  7. 检查集群文件读写是否正常；

metanode + datanode 节点混合故障模拟；

测试metanode rocksdb 引擎chubaofs集群metanode,datanode节点同时故障时,集群功能是否正常..

测试步骤:

  1. 部署 chubaofs rockdb 待测试版本集群；
  2. metanode 节点配置文件中storeType:1, 设置所有 metanode 存储引擎为 rocksdb 存储模式；

压力测试

测试metanode rocksdb 引擎chubaofs集群在高负载时的功能性能.

海量文件并发读写

测试metanode rocksdb 引擎chubaofs集群海量(亿级)文件并发读写时的功能性能是否正常.

测试步骤:

  1. 部署 chubaofs rockdb 待测试版本集群；
  2. metanode 节点配置文件中storeType:1, 设置所有 metanode 存储引擎为 rocksdb 存储模式；
  3. 启动 chubaofs 集群;
  4. 挂载 vol(若无 vol，则创建一个)；
  5. 在 vol 中创建 10 亿个待测试文件；
  6. 对待测试文件进行并发读写测试, 检查读写是否正常；

多客户端单 metapartition 并发读写

测试metanode rocksdb 引擎chubaofs集群多客户端单个metapartition并发读写时的功能性能.

测试步骤:

  1. 部署 chubaofs rockdb 待测试版本集群；
  2. metanode 节点配置文件中storeType:1, 设置所有 metanode 存储引擎为 rocksdb 存储模式；
  3. 启动 chubaofs 集群;
  4. 创建一个 vol, 设置 vol metapartition 个数为 1;
  5. 使用多个 client 同时挂载该 vol;
  6. 同时对这多个vol启动读写测试;

迁移测试

测试metanode rocksdb 引擎chubaofs集群删除文件是否正常.

新增节点

在已有 metanode memory 集群新增 rocksdb metanode 节点测试；

测试步骤:

  1. 配置chubaofs集群, metanode使用非metanode rocksdb版本, 并启动集群,进行正常读写;
  2. 在测试集群中, 新增几个rocksdb metanode节点, 启动新增rocksdb metanode节点;
  3. 运行一段时间,测试集群读写是否正常;

memory节点下线

metanode(memory + rocksdb) 混合集群 memory 节点下线测试；

测试步骤:

  1. 配置chubaofs集群, 其中部分metanode节点使用memory版本, 部分metanode节点使用rocksdb版本;

  2. 挂载客户端,对集群进行读写;

  3. 选择memory metanode节点, 执行下线操作;

  4. 观察集群是否正常;

metanode(memory + rocksdb) 集群 rocksdb 节点下线测试；

测试步骤:

测试chubaofs集群metanode rocksdb 引擎运行不同应用的功能性能.

elasticsearch

测试es 在 chubaofs集群metanode rocksdb 引擎上的功能性能.

测试步骤:

  1. 部署并启动待测试chubaofs server集群;

  2. 创建测试vol,并在部署es节点机器上启动chubaofs-client, 挂载测试vol到es 数据目录;

  3. 将es 配置文件$ES_HOME/config/elasticsearch.yml中path.data 设置为chubaofs-client挂载的测试目录(或子目录);

  4. 启动es集群;

  5. 配置forcebot es测试用例es集群连接地址到该es测试集群;

  6. 启动forcebot es 测试用例,执行测试;

  7. 随机重启chubaofs metanode/datanode节点, 观察es读写情况;
